# Best2 앱 고도화 계획 (Next Steps) - v4.0

## 1. 핵심 목표: "Reactive Tool"에서 "Proactive Service"로 진화
- **현재:** "사용자가 사진을 찍으면" -> "알아서 정리해준다" (수동적/반응형)
- **미래:** "심심할 때 앱을 켜면" -> "정리할 거리를 던져주고, 즐기게 한다" (능동적/제안형)
- **가치:** 단순한 도구를 넘어, 나의 소중한 추억을 관리해 주는 **"AI 갤러리 매니저"**

---

## 2. UI/UX 개편: "청소의 게임화(Gamification)"

홈 화면을 단순한 대시보드가 아닌, 사용자가 참여하고 싶게 만드는 **Activity Hub**로 전환합니다.

### 2.1. 홈 화면 (My Gallery Center)
- **상단 (Status Bar):** "오늘의 갤러리 상태: **쾌적함** 🌿"
    - 정리된 사진 비율이나 남은 용량에 따라 멘트와 아이콘(이모지)을 동적으로 변경하여 감성적인 피드백 제공.
- **중앙 (Mission Card - 핵심):**
    - 상황에 맞는 정리 미션을 카드 형태로 제안.
    - 예: *"3년 전 오늘, 제주도 여행 사진이 300장이나 쌓여있어요! 10장으로 줄여볼까요?"*
    - 예: *"스크린샷이 50장 쌓였습니다. 유통기한 지난 정보를 정리할까요?"*
- **하단 (Quick Action):**
    - **[📸 오늘 정리]:** 기존 '분석 시작' 버튼.
    - **[🧹 스크린샷 청소]:** 스크린샷 전용 관리 모드 진입.
    - **[☁️ 클라우드 팁]:** 기존 PC 활용 가이드(AI Toss).

### 2.2. 리뷰 화면 (Selection Fun)
- **틴더(Tinder) 스타일 리뷰 (옵션):**
    - '나머지' 사진들을 리스트 대신 한 장씩 카드로 보여줌.
    - **왼쪽 스와이프:** 삭제 (Trash)
    - **오른쪽 스와이프:** 살리기 (Keep)
    - 지루한 검토 과정을 가벼운 놀이처럼 변경하여 피로도 감소.

## 3. 핵심 기능 확장: "맥락(Context)을 이해하는 정리"

### 3.1. 과거 정리 (Legacy Cleaning) - **[우선순위 1]**
- **"N년 전 오늘" (On this day):**
    - 구글 포토의 '추억' 기능처럼, 과거의 오늘 날짜에 찍은 사진을 불러와 정리 제안.
    - 사용자가 잊고 있던 추억을 소환함과 동시에 정리까지 유도.
- **이벤트(Event) 기반 정리:**
    - 특정 기간에 사진 촬영이 급증한 구간(여행, 결혼식, 모임 등)을 자동 감지하여 "지난주 '강릉 여행' 정리하기" 제안.

### 3.2. 스마트 클리너 (Smart Cleaner) - **[우선순위 2]**
- **스크린샷 수명 관리:**
    - 단순히 제외하는 것을 넘어, "오래된 스크린샷"을 찾아냄.
    - 캡처한 지 1주일/1달 지난 사진, 텍스트만 있는 배송 조회 캡처 등을 감지하여 일괄 삭제 유도.
- **중복/유사 사진 초고속 삭제:**
    - pHash가 완전히 동일하거나(Duplicate), 거의 유사한(Burst) 사진 그룹을 찾아내어, "이 중 베스트 1장만 남기기" 제안.

### 3.3. 분석 로직 고도화 (AI Brain Upgrade)**
- **감점 로직 강화:**
    - 현재의 '찡그림(-10점)'에 더해, '흔들림(Motion Blur)'과 '역광' 감지 시 감점 로직 추가.
- **문서/영수증 인식:**
    - ML Kit Image Labeling을 활용하여 'Document', 'Receipt' 등을 감지.
    - 인물/풍경 사진과 분리하여 별도 관리하거나, 텍스트 선명도를 기준으로 평가.

## 4. 추천 개발 로드맵 (Roadmap)

가장 현실적이고 사용자 만족도가 높은 순서로 제안합니다.

### Phase 1: "추억 소환과 정리" (The Past)
1.  **기능:** **"N년 전 오늘 사진 정리하기"** 기능 구현.
2.  **UI:** 홈 화면 중앙에 '과거 정리 미션 카드' 배치.
3.  **Backend:** 기존 `ClusteringWorker`를 재활용하되, 입력 소스를 '특정 날짜 범위'로 지정할 수 있도록 파라미터화.

### Phase 2: "지능형 청소부" (The Cleaner)
1.  **기능:** **"오래된 스크린샷 감지 및 삭제"** 기능 구현.
2.  **UI:** 홈 화면 하단에 '스크린샷 청소' 퀵 버튼 추가.
3.  **Backend:** 갤러리 전체 스캔 로직 최적화 (미디어 타입 필터링).

### Phase 3: "즐거운 리뷰 경험" (The Play)
1.  **기능:** 리뷰 화면에 **스와이프(Swipe) 모드** 도입.
2.  **UI:** 애니메이션과 햅틱 피드백을 추가하여 손맛이 있는 정리 경험 제공.

## 5. 기술적 고려사항 (Tech Note)

- **확장성 (Scalability):** `PhotoAnalysisWorker`와 `ReviewViewModel`이 '오늘 찍은 사진'뿐만 아니라 '임의의 사진 리스트'를 처리할 수 있도록 유연하게 설계 변경 필요.
- **권한 (Permission):** 과거 사진(Android 10+ Scoped Storage) 접근 및 삭제 시 사용자 권한 요청 흐름(ActivityResult)을 매끄럽게 처리해야 함.
- **성능 (Performance):** 수천 장의 과거 사진을 스캔할 때 UI가 버벅이지 않도록 페이징(Paging) 처리 및 백그라운드 로직 최적화 필수.
